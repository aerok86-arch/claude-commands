# /book-summary — 책 심층 요약

호출 즉시 시작. 확인 질문 없이 자율 진행.

---

## STEP 0 — 입력 감지

`$ARGUMENTS`를 확인해 분기:

| 조건 | 처리 |
|---|---|
| `.pdf` 확장자 | PDF 분기 → Read(pages 단위) 또는 PDF MCP 도구 |
| `.txt` / `.md` / `.epub` / `.docx` | 텍스트 분기 → Read |
| `http://` 또는 `https://` | URL 분기 → WebFetch |
| 긴 텍스트 붙여넣기 | 직접 텍스트 분기 |
| 비어 있음 | 사용자에게 입력 방식 묻기 (1회만) |

---

## STEP 1 — 전체 텍스트 수집

### PDF 처리
- `Read` 도구로 읽기 시도. 20페이지 초과 시 `pages` 파라미터로 청크 분할 (최대 20페이지씩).
- 예: 300페이지 책 → "1-20", "21-40", ..., "281-300" 으로 순차 읽기.
- 각 청크 요약을 누적 → 최종 통합.

### 텍스트 파일
- `Read`로 전체 읽기. 2000줄 초과 시 offset/limit으로 분할 읽기.

### URL
- `WebFetch`로 수집. 책 소개 페이지라면 공개된 내용 최대한 활용.
- 아마존/교보/예스24 등 판매 페이지 → 목차·설명·저자 소개 수집.

### 직접 붙여넣기
- 그대로 사용.

## 품질 기준

- **장별 요약**은 가장 핵심. 절대 "이 장은 X를 다룬다" 수준으로 끝내지 않는다. 구체적 논거, 사례, 데이터를 포함.
- **핵심 테제**는 저자 관점을 정확히 반영. 내 의견이 아님.
- **인사이트**는 독자(PE 운용역, 투자 전문가) 관점에서 실용적으로.
- 요약본 길이 제한 없음. 500페이지 책이면 10,000자 요약도 무방.
- 텍스트 접근 제한(페이월, DRM)이면: 공개 정보(서문, 목차, 리뷰, 저자 인터뷰)를 최대한 활용하고 한계를 명시.
