# session-handoff — 세션 간 작업 이관

이 세션이 직접 실행할 수 없는 환경(다른 컴퓨터의 로컬 파일시스템, 다른 OS, 다른 사람의 세션 등)에서
작업을 마무리해야 할 때, "받는 쪽 Claude Code 세션"이 이 세션과 동등한 맥락을 가지고 자율적으로
이어서 실행할 수 있도록 패키징하는 스킬. 단순히 "이렇게 해줘"라고 말로 전달하는 대신, 검증 가능하고
재실행 가능한 형태(문서 + 선택적 슬래시커맨드)로 남긴다.

## 언제 barrier가 되는가

이 스킬은 받는 쪽 세션이 **이 세션이 접근 못 하는 환경**에서 실행해야 할 때 쓴다 (다른 머신의 파일시스템,
다른 OS라 스크립트 문법이 다름, 다른 사람의 인증정보/MCP 연결이 필요함 등). 단순히 "나중에 계속하자"는
동일 환경 내 재개는 이 스킬 대상이 아니다 — 그건 그냥 대화를 이어가거나 `/checkpoint`류를 쓰면 된다.

## 절차

### 1. 대상(target) 파악
누구/어디로 넘기는지 확인한다 — 사용자가 이미 말했으면(예: "윈도우 PC") 그대로 쓰고, 불명확하면 물어본다:
- 다른 컴퓨터(OS 다름 → 스크립트/명령어 이식 필요)
- 같은 컴퓨터의 다른 세션/사람
- 접근 방법: git repo를 clone/pull 할 수 있는지, 아니면 파일을 수동으로 옮겨야 하는지

### 2. 현재 세션 상태 요약
아래를 빠짐없이 정리한다 — 받는 세션은 이 대화를 전혀 보지 못하므로, 여기 없는 정보는 존재하지 않는 것과 같다:
- **목표**: 원래 무엇을 하려던 작업인지, 왜(배경)
- **이미 한 일**: 변경된 파일, 커밋, 확인된 사실 (구체적 경로·커밋해시 포함)
- **결정사항과 이유**: 여러 방법 중 왜 이 방법을 택했는지 (예: "API 과금 방식은 사용자가 거부해서 대신 구독 기반 CLI 유지")
- **제약조건/하지 말아야 할 것**: 받는 세션이 실수하기 쉬운 함정 (예: "ANTHROPIC_API_KEY 설정하지 말 것", "프로덕션에 확인 없이 배포 금지")
- **남은 작업**: 순서대로, 받는 세션이 판단해야 할 지점은 "직접 검증/질문하라"고 명시(내가 검증 못 하는 부분을 마치 확실한 것처럼 쓰지 않는다)
- **완료 후 보고 형식**: 받는 세션이 끝나고 사용자에게 뭘 어떻게 보고해야 하는지

### 3. 문서 위치 결정
- **git repo 안의 작업이면**: 그 repo 안에 문서를 둔다(예: 프로젝트에 이미 운영/README 로그가 있으면 거기 이어서 쓰고, 없으면 `<주제>_HANDOFF.md` 신규 생성). git으로 clone/pull하면 자동으로 같이 넘어가므로 별도 전달 수단이 필요 없다.
- **git repo가 아니거나 받는 쪽이 repo 접근이 안 되면**: 사용자에게 전달 방법을 물어본다(공유 드라이브, 직접 붙여넣기 등) — 임의로 추측해서 진행하지 않는다.
- **절대 금지**: 토큰·비밀번호·API 키 등 민감정보를 문서에 직접 적지 않는다. "어디서 구할 수 있는지"(예: "Vercel 대시보드 → Storage → Blob → Connect")만 적고, 받는 세션이 실행 중에 사용자에게 직접 물어보게 한다.

### 4. (선택) 슬래시커맨드 진입점 생성
받는 세션에서 한 번의 명령으로 시작할 수 있게 하고 싶으면, 프로젝트의 `.claude/commands/<이름>.md`에
얇은 진입점을 만든다 — 내용을 중복하지 말고 "N번 문서를 읽고 순서대로 실행해라"만 지시한다. 이렇게 하면
받는 쪽이 git clone 직후 `claude` 세션에서 `/<이름>`만 치면 시작된다.

### 5. 커밋 + push
git repo라면 커밋해서 push한다(원격에 반영되는 작업이므로 사용자에게 간단히 확인 후 진행).

### 6. 사용자에게 전달할 짧은 안내
채팅으로 아래 형태를 준다:
- 받는 쪽에서 뭘 어떻게 시작하면 되는지 (슬래시커맨드 만들었으면 그 이름, 아니면 붙여넣을 지시문 한 단락)
- 중간에 받는 쪽이 사용자에게 물어볼 만한 것(자격증명 등)이 있다면 미리 귀띔

## 예시 (2026-07-14, pe-trendchaser 프로젝트)
맥북 launchd로 돌던 pe-knowledge 야간 처리를 24/7 Windows PC로 옮길 때: repo에 `automation/pe-knowledge/WINDOWS_SETUP.md`(맥락·제약조건·체크리스트·완료보고 형식)를 작성 → `.claude/commands/pe-knowledge-windows-setup.md`(얇은 진입점, "WINDOWS_SETUP.md를 읽고 순서대로 실행해라"만 지시) → 커밋·push → 사용자에게는 "저장소 clone 후 `claude` 세션에서 `/pe-knowledge-windows-setup` 입력"만 안내.
